Issue Tracker API Documentation

Base URL

https://your-api-domain.com/api/v1

Authentication

All API requests require authentication using a Bearer token in the Authorization header:

Authorization: Bearer your-token-here

Endpoints

Issues

List Issues

GET
/issues

Query Parameters:

  • status (string): Filter by status
  • priority (string): Filter by priority
  • assign_to (string): Filter by assigned user ID
  • created_by (string): Filter by creator ID
  • search (string): Search in title and description
  • sort_by (string): Field to sort by
  • order (string): asc or desc

Create Issue

POST
/issues
{ "title": "Issue title", "description": "Issue description", "type": "bug", "severity": "high", "priority": "high", "status": "open", "deadline": "2024-03-01", "assigned_to": ["user_id_1", "user_id_2"], "watchers": ["user_id_3", "user_id_4"] }

Get Issue

GET
/issues/{id}

Update Issue

PUT
/issues/{id}
{ "title": "Updated title", "description": "Updated description", "type": "feature", "severity": "medium", "priority": "medium", "status": "in_progress", "deadline": "2024-03-15", "assigned_to": ["user_id_1"], "watchers": ["user_id_2"] }

Delete Issue

DELETE
/issues/{id}

Bulk Create Issues

POST
/issues/bulk
[ { "title": "Issue 1", "description": "Description 1", "type": "bug", "severity": "high", "priority": "high", "status": "open" }, { "title": "Issue 2", "description": "Description 2", "type": "feature", "severity": "medium", "priority": "medium", "status": "open" } ]

Attachments

Add Attachment

POST
/issues/{id}/attachments

Content-Type: multipart/form-data

Parameters:

  • file (file): The file to upload

List Attachments

GET
/issues/{id}/attachments

Delete Attachment

DELETE
/issues/{id}/attachments/{attachment_id}

Comments

Add Comment

POST
/issues/{id}/comments
{ "text": "Comment text" }

List Comments

GET
/issues/{id}/comments

Settings

Get Settings

GET
/settings

PUT

Users

Get User Profile

GET
/users/{id}

Update User Profile

PUT
/users/{id}
{ "bio": "User bio", "avatar": "base64_encoded_image" }

Get User Issues

GET
/users/{id}/issues

Query Parameters:

  • type (string): assigned or watched
  • status (string): open or closed
  • sort_by (string): Field to sort by
  • order (string): asc or desc

Get User Comments

GET
/users/{id}/comments
